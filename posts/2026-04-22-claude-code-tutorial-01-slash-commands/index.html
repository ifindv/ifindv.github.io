斜杠命令是 Claude Code 的基础交互方式。输入 / 即可看到全部内置命令，支持自定义扩展。

内置命令速查
#

Claude Code 内置 60+ 命令，按用途分类：

日常操作
#

命令用途
/init初始化项目，生成 CLAUDE.md
/memory编辑内存文件
/clear清除对话（别名：/reset）
/exit退出

代码相关
#

命令用途
/diff查看未提交更改的 diff
/plan进入规划模式，分析任务
/rewind回溯对话或代码状态
/compact压缩上下文，节省 token

设置与信息
#

命令用途
/model切换模型（Haiku/Sonnet/Opus）
/cost查看 token 使用统计
/config打开设置
/context可视化查看上下文占用

完整列表在 Claude Code 中输入 / 查看。

自定义命令
#

自定义斜杠命令已并入技能系统。创建技能后可直接用 /技能名 调用。

目录结构
#

.claude/skills/my-command/
└── SKILL.md

基本格式
#

---
name: my-command
description: 命令功能描述，包含触发关键词
---

# 命令标题

分步骤的执行说明。

核心参数
#

参数说明
name命令名称，用 /name 调用
description最重要 — 告诉 Claude 何时使用此命令
allowed-tools无需审批可用的工具列表
disable-model-invocationtrue 时仅用户可调用，防止 Claude 自动执行

变量与动态内容
#

变量说明
$ARGUMENTS调用时传入的全部参数
$0, $1位置参数（从0开始）
!`command`执行命令并插入输出结果
@file插入文件内容到指令中

示例： 自动获取 git 状态：

---
name: git-status
description: 查看项目 git 状态
---

## 当前状态
!`git status -s`

## 未提交更改
!`git diff --stat`

调用方式
#

# 直接调用
/memory

# 带参数调用
/plan 重构用户认证模块

# 连续调用多个命令
/plan 分析代码  ->  /compact 精简上下文

最佳实践
#

  • ✅ 描述要具体：包含触发关键词，比如 “代码审查”、“安全检查”
  • ✅ 命令名要清晰：/fix-bug 比 /fix 更好
  • ✅ 危险操作加锁：disable-model-invocation: true 防止意外执行
  • ✅ 组织相关命令：用目录结构管理大量命令
  • ❌ 不硬编码敏感信息：密码、API 密钥等不要写在命令里
  • ❌ 不追求大而全：一个命令做一件事

实用场景示例
#

1. 代码审查命令
#

---
name: review
description: 审查代码质量和安全问题
---

审查最近修改的代码，关注：
1. 安全漏洞（注入、XSS、认证问题）
2. 性能问题（N+1查询、内存泄漏）
3. 代码可读性（命名、注释、复杂度）

2. 提交规范命令
#

---
name: commit
description: 生成符合规范的提交信息
allowed-tools: Bash
---

生成符合 Conventional Commits 规范的提交信息：
!`git diff --staged`

总结
#

斜杠命令是 Claude Code 最基础的交互入口。常用命令记不住没关系，输入 / 随时查看完整列表。自定义命令通过技能系统扩展，核心是写好 name 和 description，合理配置 allowed-tools 和 disable-model-invocation。
